# Архитектура системы CryptoMarketAnalysis

## Назначение системы

CryptoMarketAnalysis — система анализа криптовалютного рынка, разработанная в рамках практики Университета ИТМО.

Система получает исторические данные из открытых источников, сохраняет их в PostgreSQL, выполняет аналитическую обработку и формирует отчеты.

## Основные направления анализа:

* анализ изменения цены;
* анализ волатильности;
* корреляционный анализ;
* анализ объемов торгов;
* формирование PDF-отчетов;
* накопление исторических рыночных данных.

## Слои системы

### Domain

Содержит бизнес-сущности и бизнес-правила.

Domain не зависит ни от каких внешних библиотек.

Основные сущности:

* Asset
* MarketDataPoint
* MarketDataSource

Domain является центром системы.

---

### Application

Содержит бизнес-сценарии системы.

Использует только абстракции.

Основные use cases:

* LoadMarketDataUseCase
* AnalyzePriceChangeUseCase
* AnalyzeVolatilityUseCase
* AnalyzeCorrelationUseCase
* GenerateMarketReportUseCase

Application не зависит от Infrastructure.

---

### Infrastructure

Содержит технические реализации.

#### Основные компоненты:

Persistence

* PostgreSQL
* Entity Framework Core
* Репозитории

External APIs

* CoinGecko provider
* Binance provider

Reporting

* PDF generation
* SVG chart generation

Infrastructure реализует контракты Application.

---

### API

REST API интерфейс системы.

Использует Application layer.

#### Предоставляет:

* загрузку данных;
* аналитику;
* генерацию отчетов.

---

### CLI

Консольный интерфейс системы.

Использует Application use cases.

Поддерживает:

* load
* load backfill
* analytics
* report pdf

CLI не содержит бизнес-логики.

---

### Worker

Фоновый сервис.

Предназначен для периодической автоматической загрузки рыночных данных.

На текущем этапе присутствует как отдельный проект и будет расширяться в следующих этапах разработки.

## Источники данных

### Binance

Используется для получения исторических рыночных данных.

Особенности:

* не требует API Key;
* высокая доступность;
* подходит для массовой загрузки исторических данных;
* используется как основной источник для backfill.

---

### CoinGecko

Используется для получения рыночных данных и капитализации.

Особенности:

* требует Demo API Key;
* предоставляет данные по капитализации;
* имеет ограничения по rate limit.

## Аналитические сценарии

### Price Change Analysis

Расчет:

* начальной цены;
* конечной цены;
* абсолютного изменения цены;
* процентного изменения цены.

---

### Volatility Analysis

Расчет:

* доходностей;
* среднего значения доходностей;
* стандартного отклонения доходностей.

Текущая реализация не выполняет annualization.

---

### Correlation Analysis

Расчет коэффициента корреляции Пирсона между двумя активами.

Корреляция рассчитывается по доходностям, а не по абсолютным ценам.

---

### PDF-отчеты

PDF-отчет формируется в Infrastructure.

Содержит:

* метаданные;
* аналитическую сводку;
* график цены;
* график объема торгов;
* историческую таблицу;
* пояснения.

Для больших периодов историческая таблица автоматически сокращается.

---

### Массовая загрузка данных (Backfill)

Backfill выполняется через CLI.

Причины:

* длительное выполнение;
* отображение прогресса;
* поддержка batching;
* учет ограничений внешних API;
* отсутствие долгих HTTP-запросов.

Период разбивается на батчи.

Каждый батч обрабатывается через существующий LoadMarketDataUseCase.

## Архитектурные ограничения

Запрещено:

* обращаться из CLI напрямую к DbContext;
* обращаться из CLI напрямую к репозиториям;
* обращаться из CLI напрямую к provider-ам;
* размещать бизнес-логику в CLI;
* генерировать PDF в Application;
* использовать QuestPDF в Application;
* использовать SVG/chart generation в Application;
* изменять Domain ради инфраструктурных задач.

Разрешено:

* добавлять новые MarketDataProvider;
* добавлять новые аналитические use cases;
* расширять PDF-отчеты;
* расширять CLI;
* расширять API.

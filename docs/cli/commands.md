# CryptoMarketAnalysis CLI

Консольный интерфейс для работы с системой анализа криптовалютного рынка.

---

# Загрузка данных

## load

Загружает исторические рыночные данные через зарегистрированные providers.

Пример:

```bash
crypto load \
  --symbols BTC \
  --symbols ETH \
  --from 2026-06-01 \
  --to 2026-06-10 \
  --source BINANCE
```

Параметры:

| Параметр | Описание |
|-----------|-----------|
| --symbols | Символ актива |
| --from | Начало периода |
| --to | Конец периода |
| --source | Источник данных |

Результат:

- количество загруженных точек;
- количество пропущенных дублей;
- итоговая сводка.

---

# Аналитика

## analytics price-change

Расчет изменения цены.

Пример:

```bash
crypto analytics price-change \
  --symbol BTC \
  --from 2026-06-01 \
  --to 2026-06-10 \
  --source BINANCE
```

Вывод:

- начальная цена;
- конечная цена;
- абсолютное изменение;
- процентное изменение.

---

## analytics volatility

Расчет волатильности.

Пример:

```bash
crypto analytics volatility \
  --symbol BTC \
  --from 2026-06-01 \
  --to 2026-06-10 \
  --source BINANCE
```

Вывод:

- количество точек;
- количество доходностей;
- средняя доходность;
- волатильность.

---

## analytics correlation

Расчет корреляции двух активов.

Пример:

```bash
crypto analytics correlation \
  --base BTC \
  --quote ETH \
  --from 2026-06-01 \
  --to 2026-06-10 \
  --source BINANCE
```

Вывод:

- количество совпавших доходностей;
- коэффициент корреляции Пирсона.

# Отчеты

## report pdf

Генерация PDF-отчета.

Пример:

```bash
crypto report pdf \
  --symbol BTC \
  --from 2026-06-01 \
  --to 2026-06-10 \
  --source BINANCE \
  --correlation ETH \
  --output ./reports/btc-report.pdf
```

Параметры:

| Параметр | Описание |
|-----------|-----------|
| --symbol | Символ актива |
| --from | Начало периода |
| --to | Конец периода |
| --source | Источник данных |
| --correlation | Второй актив для корреляции |
| --output | Путь сохранения PDF |

Результат:

- создание PDF-файла;
- отображение размера файла;
- отображение количества точек анализа.

---

# Архитектура

CLI:

- использует Application Use Cases;
- не работает напрямую с DbContext;
- не работает напрямую с репозиториями;
- не работает напрямую с CoinGecko или Binance API;
- не содержит бизнес-логику аналитики;
- не содержит логику генерации PDF.
